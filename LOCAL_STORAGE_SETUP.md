## Локальное хранилище аватарок и баннеров

Проект настроен на локальное сохранение аватарок и баннеров пользователей вместо Supabase.

### Структура папок
```
img/
├── avatars/      # Аватарки пользователей
└── banners/      # Баннеры пользователей
```

### Как запустить

1. **Установи зависимости:**
```bash
npm install
```

2. **Запусти проект в режиме разработки:**
```bash
npm run dev
```

Это автоматически запустит:
- Express сервер на порту 5000 (обработка загрузок)
- Vite dev сервер на порту 5173 (фронтенд)

### Как работает

1. Пользователь выбирает аватарку или баннер в Settings
2. Файл отправляется на `/api/upload`
3. Express сервер сохраняет файл в `img/avatars` или `img/banners`
4. Возвращается путь вида `/img/avatars/user-id.jpg`
5. Путь сохраняется в БД (таблица `profiles`)
6. Изображение отображается с локального хранилища

### Ограничения

- Максимум 10MB для одного файла
- Поддерживаемые форматы: JPEG, PNG, GIF, WebP
- Файл перезаписывается при повторной загрузке (сохраняется по ID пользователя)

### Дополнительно

- Папка `img/` добавлена в `.gitignore` (изображения не коммитятся в git)
- На продакшене убедись, что папка `img/` существует и доступна серверу
- Можешь добавить регулярную чистку старых файлов или установить лимит на их хранение
